## Required test coverage by surface

### 1. Claim flow (HIGHEST RISK)
Identity + ownership + takedown — legal exposure if broken.

- **Unit:** magic-link token generation, expiry (24h), single-use enforcement
- **Integration:** unsolicited locker → claim email sent → token consumed → ownership transfer → original draft author loses write access
- **E2E:** athlete clicks email link on mobile Safari → lands on locker → can edit bio → can request takedown → takedown removes from public index within 60s
- **Adversarial:** two athletes claim same locker (collision); expired token replay; token leaked in URL referer

### 2. Provenance + consent (LEGAL CRITICAL)
- **Unit:** `Clip.source_provenance` enum constraint rejects unknown sources
- **Integration:** every Clip insert requires a linked Consent row; orphaned clips fail validation
- **UI:** provenance badge renders on every clip surface (locker page, OG image, share card) — snapshot test
- **Audit:** query "show all clips with source=cal_archive" returns the consent record for each

### 3. Auto-draft pipeline
- **Unit:** each scraper (YouTube, Wiki, ESPN, school, NFL) handles 404, rate limit, malformed HTML
- **Integration:** name+school+position → draft Locker with bio + ≥3 timeline entries + ≥1 clip candidate, status=pending_founder_review
- **Quality gate:** Claude-generated bio passes a "no hallucinated stats" check (cross-reference numeric claims against scraped sources; flag mismatches for founder)
- **Wrong-person guard:** ambiguous name (e.g., two "John Smith QB") routes to founder queue with disambiguation UI, never auto-publishes

### 4. Founder approval queue
- **Unit:** queue ordering (FIFO), status transitions (pending → approved | rejected)
- **Integration:** approval flips locker to athlete-visible (not yet public); rejection notifies pipeline to retry or discard
- **Auth:** only founder role can access /admin/queue

### 5. Athlete first-view "magic moment"
- **Perceived perf:** from athlete claim-link click to "your locker is ready" reveal ≤ 30s p95
- **Visual:** confetti / reveal animation fires once per locker
- **Edit:** athlete can edit bio, add clips, reorder timeline; changes persist

### 6. Public locker page
- **SEO:** server-rendered, lighthouse SEO score ≥ 95
- **OG image:** generated per locker, includes hero clip thumb + name + position + provenance footer
- **Share:** iMessage, Twitter, LinkedIn previews render correctly (manual checklist)
- **Follow:** one-click follow captures email; digest job sends weekly

### 7. Video pipeline (Mux)
- **Integration:** upload → webhook → asset ready → playable HLS URL stored on Clip row
- **Failure:** upload >2GB rejected with user-visible error; transcode failure surfaces in admin queue

## Out of scope for MVP test suite
- Payments, Stripe, payout splits (P5 deferred)
- Coach dashboard
- NIL compliance engine
- Auto-highlight ML (manual cuts only at MVP)
- Mobile native

## Test infrastructure recommendations

- **Framework:** Vitest (unit + integration) + Playwright (E2E)
- **DB:** Supabase local stack via `supabase start` for integration tests
- **Mux:** sandbox env + recorded fixture responses for CI
- **Claude:** mocked in CI (canned responses); live calls only in nightly quality-gate job
- **CI gates:** PR must pass unit + integration; E2E + lighthouse on preview deploy

## Coverage targets at week 6 (50 lockers live)

- Claim flow: 100% branch coverage (legal critical)
- Provenance/consent: 100% branch coverage (legal critical)
- Auto-draft pipeline: ≥80% line coverage, golden-file tests for 5 known athletes
- Public locker page: visual regression baseline established
- E2E happy path: claim → edit → publish → share runs green on every PR
